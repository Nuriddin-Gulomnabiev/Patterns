Задача: Система отчётов для международной компании

Компания работает в нескольких странах. Для каждой страны у отчётов свои особенности:

Разные форматы документов: PDF, Excel, HTML.

Разные локализации: валюты, даты, язык.

Разные правила отображения данных (например, в США суммы с $, а в Европе — с €).

Условия:

У тебя должен быть клиентский код (например, ReportService), который не должен знать, в какой стране запускается система.

Для каждой страны создаётся своя фабрика, которая генерирует семейство объектов:

IReportGenerator (разные форматы: PDF, Excel, HTML)

ILocalizationProvider (разные валюты, даты, языки)

IDataFormatter (разные правила форматирования чисел/сумм/дат).

Требования:

Реализовать фабрики для США и Европы.

Клиентский код должен получать фабрику в конструкторе и работать одинаково для любой страны.

Подумать, как будешь расширять систему, если завтра появятся фабрики для Японии или Ближнего Востока.

Добавь хотя бы один сценарий, где внутри семейства продуктов объекты реально связаны между собой (например, ILocalizationProvider влияет на работу IReportGenerator).
